# Firebase Shared List Schema (Current Implementation)

This document reflects the schema that is currently written and read by the app in `index.html` and `script.js`.

## Realtime Database Path

All shared data is stored under:

```text
sharedLists/{code}
```

- `{code}`: 6-character share code (A-Z, 0-9).

## Current Stored Shape

```text
sharedLists
  {code}
    title: string
    updatedAt: number (epoch milliseconds)
    tasks: Task[]
```

```text
Task
  text: string
  done: boolean
  photos: Photo[]
```

```text
Photo
  id: string
  dataUrl: string
  createdAt: number (epoch milliseconds)
```

## Example

```json
{
  "sharedLists": {
    "A1B2C3": {
      "title": "Weekend Chores",
      "updatedAt": 1768425600000,
      "tasks": [
        {
          "text": "Clean the kitchen",
          "done": false,
          "photos": [
            {
              "id": "p_lm9xk2_j8na2c",
              "dataUrl": "data:image/jpeg;base64,/9j/4AAQSk...",
              "createdAt": 1768425600000
            }
          ]
        },
        {
          "text": "Buy groceries",
          "done": true,
          "photos": []
        }
      ]
    }
  }
}
```

## Important Notes

- Task IDs are local-only in this app and are not stored in the shared payload.
- `updatedAt` is generated by the client at write time.
- The app sanitizes tasks/photos before writing and while reading remote data.

## Limits Applied by Client

- Max photos per task in shared payload: `4`.
- Max `dataUrl` length per shared photo: `400000` characters.
- Max serialized payload target: `7 MB` (the client trims largest photos first to stay under this cap).

## Not Used in the Current Flow

- Firebase Storage is not used by the current client flow.
- Firestore is not used by the current client flow.
- Firebase Auth is not used by the current client flow.

If Storage metadata (`mimeType`, `width`, `height`) is introduced later, this document must be updated together with client serialization and database rules.

## Security Recommendation (Operational)

Even with client-side validation, enforce Realtime Database Rules for:

- Read/write access to `sharedLists/{code}`.
- Required field types (`title`, `updatedAt`, `tasks`, `photos`).
- Basic code-path constraints and payload validation where possible.
